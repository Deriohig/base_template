Working at IBM

Challenges

Enterprise and scale

Working in an enterprise company has taught me the importance of scale and reusablity. Not only when itterating and developing features, but also in scaling my knowledge and expertise to the wider team. Creating patterns that can be leveraged across many diciplines is a huge challenge I am trying to improve upon every day.

Technologies

During my time in IBM I've worked on two Security products. The first was an Angular 8 data classification application running on a microservice architecture. And most recently, a next-gen data security suite built using React with server side rendering framework Next.js. Both applications ran using a REST API, and leveraged the use of IBM's Carbon components and the IBM Design Language.

Distributed team

I've worked within a distributed team throughout my time at IBM, working across 4 timezones (GMT, Eastern, Central, Pacific). Altought working within a co-located team has obvious operational benefits, working in a distributed team has enabled me to learn more about the importance of sprint, backlog grooming, and thourough planning.

The Digital Hub

The digital hub is a website I worked as one of two front end developers, I was responsible for the dashboard of the members area of the site as well as some components of the homepage including the masonry grid. Technologies used were HTML, SCSS, JavaScript, PHP, and Wordpress.
