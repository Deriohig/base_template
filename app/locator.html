Working on the locator

The locator was the first project I worked on in Software Design. Built for providing pickup point locations for DPD, a delivery service competetor to DHL and FedEx.

The locator works by combining an idependent API alongside the google maps API in order to map locations based on lat long co-ordinates, provide detailed information for locations, get directions to locations, find your nearest location, filter locations by services, search by text, and more.

The first locator was used for DPD, but it was redeveloped for Merlyn Showering who had different requirements in both layout and functionality. The Locator is a jQuery plugin invoked in the typical jQuery fashion with a customisable onOpen( ) and onClose( ) method configuration. Features include pre-loaded search and filters, pre-selected locations, and model (lightbox) or inline loading.

Challenges

Communication

At times the project stalled, designs were signed off and the client wanted changes made (and mostly justifibably), but the process in place meant that everything needed to be aproved beforehand. A small change could take longer than a feature due to the back and forth between client, design team, and project manager, impacting on the estimates and deadlines. Altough this isn't something that can be completely avoided, it was important to reduce overheads and push people to make decisions.

Timeframe

The timeframe for the Merlyn project in particular was very tight. I had to implement new feature and change how existing ones worked, while doing a re-configuring of the layout. It was completed on time and the clients were very happy with the finishing product.

Keeping track of the Code Base

With a few thousand lines of code, not including the markup, it got a bit out of hand trying to organise the growing codebase while continualy implementing new features and reformatting it to fit a new project. An incorrect aproach at the begining made later changes unnecesarily painful, which was in part due to a lack of a full spec but also in part to a rushed implementation.
